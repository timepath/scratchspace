4 /*
5  List of all notifications (including identifiers and display information)
6  Possible Tokens:
7  default, name, strnum, flnum,
8  channel, sound, volume, position,
9  args, hudargs, icon, cpid, durcnt, normal, gentle,
10  anncename, infoname, centername,
11  challow, chtype, optiona, optionb
12  Format Specifications:
13  MSG_ANNCE:
14  default: FLOAT: Default setting for whether the notification is enabled or not
15  ^-> 0 = disabled, 1 = enabled if gentle is disabled, 2 = always enabled
16  name: VAR: Name of notification
17  channel: FLOAT: Sound channel to broadcast on to
18  sound: STRING: Filename for the announcement sound
19  volume: FLOAT: Volume setting for the announcement sound
20  position: FLOAT: Attenuation/positioning value
21  MSG_INFO:
22  default: FLOAT: Default setting for whether the notification is enabled or not
23  ^-> 0 = disabled, 1 = enabled, 2 = also print to chat box
24  name: VAR: Name of notification
25  strnum: FLOAT: Number of STRING arguments (so that networking knows how many to send/receive)
26  flnum: FLOAT: Number of FLOAT arguments (so that networking knows how many to send/receive)
27  args: STRING: Arguments for Local_Notification_sprintf()
28  hudargs: STRING: Arguments for Local_Notification_HUD_Notify_Push()
29  icon: STRING: icon string name for the hud notify panel, "" if no icon is used
30  normal: STRING: Normal message (string for sprintf when gentle messages are NOT enabled)
31  gentle: STRING: Gentle message (string for sprintf when gentle messages ARE enabled)
32  MSG_CENTER:
33  default: FLOAT: Default setting for whether the notification is enabled or not
34  ^-> 0 = disabled, 1 = enabled
35  name: VAR: Name of notification
36  strnum: FLOAT: Number of STRING arguments (so that networking knows how many to send/receive)
37  flnum: FLOAT: Number of FLOAT arguments (so that networking knows how many to send/receive)
38  args: STRING: Arguments for Local_Notification_sprintf()
39  cpid: FLOAT: centerprint ID number (CPID_*), NO_CPID if no CPID is needed
40  durcnt: XPD(FLOAT, FLOAT): Duration/Countdown: extra arguments for centerprint messages
41  normal: STRING: Normal message (string for sprintf when gentle messages are NOT enabled)
42  gentle: STRING: Gentle message (string for sprintf when gentle messages ARE enabled)
43  MSG_MULTI:
44  default: FLOAT: Default setting for whether the notification is enabled or not
45  ^-> 0 = disabled, 1 = enabled
46  name: VAR: Name of chaining notification
47  anncename: VAR: Name of announcer notification for reference
48  infoname: VAR: Name of info notification for reference
49  centername: VAR: Name of centerprint notification for reference
50  MSG_CHOICE:
51  default: FLOAT: Default setting for whether the notification is enabled or not
52  ^-> 0 = disabled, 1 = select option A, 2 = selection option B
53  challow: FLOAT: Default setting for server allowing choices other than A
54  ^-> 0 = no choice, 1 = allowed in warmup, 2 = always allowed
55  name: VAR: Name of choice notification
56  chtype: VAR: Notification message type for options
57  optiona: VAR: Name of choice "A" notification for reference
58  optionb: VAR: Name of choice "B" notification for reference
59 
60  Messages with ^F1, ^BG, ^TC, etc etc in them will replace those strings
61  with colors according to the cvars the user has chosen. This allows for
62  users to create unique color profiles for their HUD, giving more customization
63  options to HUD designers and end users who want such a feature.
64 
65  Check out the definitions in util.qc/util.qh/teams.qh for string CCR(...) and
66  string TCR(...) to better understand how these code replacements work.
67 
68  Additionally, you can find all the definitions and explanations for
69  the argument values and what they return down below in this file.
70 
71  Guidlines for notification declaration (please try and follow these):
72  Specific rules:
73  -ALWAYS start the string with a color, preferably background.
74  -ALWAYS reset a color after a name (this way they don't set it for the whole string).
75  -NEVER re-declare an event twice.
76  -NEVER add or remove tokens from the format, it SHOULD already work.
77  -MSG_INFO hudargs must always be ATTACKER -> VICTIM
78  -MSG_INFO and MSG_CENTER should NOT end with a new line
79 
80  General rules:
81  -Be clean and simple with your notification naming,
82  nothing too long for the name field... Abbreviations are your friend. :D
83  -Keep the spacing as clean as possible... if the arguments are abnormally long,
84  it's okay to go out of line a bit... but try and keep it clean still.
85  - Use ONLY spaces for spacing in the notification list, tabs are too inconsistent
86  with keeping alignment on different mediums.
87  -Sort the notifications in the most appropriate order for their tasks.
88  -ARIRE unir frk jvgu lbhe bja zbgure. (gvc sbe zvxrrhfn) -- Don't pay attention to this ^_^
89 
90  Final note: DO NOT PROVIDE MORE ARGUMENTS THAN NECESSARY FOR THE NOTIFICATION YOU'RE CALLING!
91  The system is designed to save as much networking bandwidth as possible,
92  so please dynamically control your argument sending to fit *exactly* what is required.
93  If you send a notification with mismatching arguments, Send_Notification() will error.
94 */
