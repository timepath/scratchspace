00004 /*
00005  List of all notifications (including identifiers and display information)
00006  Possible Tokens:
00007     default, name, strnum, flnum,
00008     channel, sound, volume, position,
00009     args, hudargs, icon, cpid, durcnt, normal, gentle,
00010     anncename, infoname, centername,
00011     challow, chtype, optiona, optionb
00012  Format Specifications:
00013     MSG_ANNCE:
00014       default: FLOAT: Default setting for whether the notification is enabled or not
00015          ^-> 0 = disabled, 1 = enabled if gentle is disabled, 2 = always enabled
00016       name: VAR: Name of notification
00017       channel: FLOAT: Sound channel to broadcast on to
00018       sound: STRING: Filename for the announcement sound
00019       volume: FLOAT: Volume setting for the announcement sound
00020       position: FLOAT: Attenuation/positioning value
00021     MSG_INFO:
00022       default: FLOAT: Default setting for whether the notification is enabled or not
00023          ^-> 0 = disabled, 1 = enabled, 2 = also print to chat box
00024       name: VAR: Name of notification
00025       strnum: FLOAT: Number of STRING arguments (so that networking knows how many to send/receive)
00026       flnum: FLOAT: Number of FLOAT arguments (so that networking knows how many to send/receive)
00027       args: STRING: Arguments for Local_Notification_sprintf()
00028       hudargs: STRING: Arguments for Local_Notification_HUD_Notify_Push()
00029       icon: STRING: icon string name for the hud notify panel, "" if no icon is used
00030       normal: STRING: Normal message (string for sprintf when gentle messages are NOT enabled)
00031       gentle: STRING: Gentle message (string for sprintf when gentle messages ARE enabled)
00032     MSG_CENTER:
00033       default: FLOAT: Default setting for whether the notification is enabled or not
00034          ^-> 0 = disabled, 1 = enabled
00035       name: VAR: Name of notification
00036       strnum: FLOAT: Number of STRING arguments (so that networking knows how many to send/receive)
00037       flnum: FLOAT: Number of FLOAT arguments (so that networking knows how many to send/receive)
00038       args: STRING: Arguments for Local_Notification_sprintf()
00039       cpid: FLOAT: centerprint ID number (CPID_*), NO_CPID if no CPID is needed
00040       durcnt: XPD(FLOAT, FLOAT): Duration/Countdown: extra arguments for centerprint messages
00041       normal: STRING: Normal message (string for sprintf when gentle messages are NOT enabled)
00042       gentle: STRING: Gentle message (string for sprintf when gentle messages ARE enabled)
00043     MSG_MULTI:
00044       default: FLOAT: Default setting for whether the notification is enabled or not
00045          ^-> 0 = disabled, 1 = enabled
00046       name: VAR: Name of chaining notification
00047       anncename: VAR: Name of announcer notification for reference
00048       infoname: VAR: Name of info notification for reference
00049       centername: VAR: Name of centerprint notification for reference
00050     MSG_CHOICE:
00051       default: FLOAT: Default setting for whether the notification is enabled or not
00052          ^-> 0 = disabled, 1 = select option A, 2 = selection option B
00053       challow: FLOAT: Default setting for server allowing choices other than A
00054          ^-> 0 = no choice, 1 = allowed in warmup, 2 = always allowed
00055       name: VAR: Name of choice notification
00056       chtype: VAR: Notification message type for options
00057       optiona: VAR: Name of choice "A" notification for reference
00058       optionb: VAR: Name of choice "B" notification for reference
00059 
00060  Messages with ^F1, ^BG, ^TC, etc etc in them will replace those strings
00061  with colors according to the cvars the user has chosen. This allows for
00062  users to create unique color profiles for their HUD, giving more customization
00063  options to HUD designers and end users who want such a feature.
00064 
00065  Check out the definitions in util.qc/util.qh/teams.qh for string CCR(...) and
00066  string TCR(...) to better understand how these code replacements work.
00067 
00068  Additionally, you can find all the definitions and explanations for
00069  the argument values and what they return down below in this file.
00070 
00071  Guidlines for notification declaration (please try and follow these):
00072   Specific rules:
00073     -ALWAYS start the string with a color, preferably background.
00074     -ALWAYS reset a color after a name (this way they don't set it for the whole string).
00075     -NEVER re-declare an event twice.
00076     -NEVER add or remove tokens from the format, it SHOULD already work.
00077     -MSG_INFO hudargs must always be ATTACKER -> VICTIM
00078     -MSG_INFO and MSG_CENTER should NOT end with a new line
00079 
00080  General rules:
00081     -Be clean and simple with your notification naming,
00082      nothing too long for the name field... Abbreviations are your friend. :D
00083     -Keep the spacing as clean as possible... if the arguments are abnormally long,
00084       it's okay to go out of line a bit... but try and keep it clean still.
00085     - Use ONLY spaces for spacing in the notification list, tabs are too inconsistent
00086       with keeping alignment on different mediums.
00087     -Sort the notifications in the most appropriate order for their tasks.
00088     -ARIRE unir frk jvgu lbhe bja zbgure. (gvc sbe zvxrrhfn) -- Don't pay attention to this ^_^
00089 
00090  Final note: DO NOT PROVIDE MORE ARGUMENTS THAN NECESSARY FOR THE NOTIFICATION YOU'RE CALLING!
00091       The system is designed to save as much networking bandwidth as possible,
00092       so please dynamically control your argument sending to fit *exactly* what is required.
00093       If you send a notification with mismatching arguments, Send_Notification() will error.
00094 */
